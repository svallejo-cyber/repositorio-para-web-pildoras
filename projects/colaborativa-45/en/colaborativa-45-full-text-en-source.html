Cuaderno de Bitácora Hub Empleo Proyecto IA + HubSpot para la gestión de candidaturas y publicación dinámica de ofertas de empleo Isaval Global

1. Contexto del proyecto Dentro del proceso de transformación digital de Isaval Global, surgió la necesidad de construir el nuevo sitio web corporativo utilizando el ecosistema de HubSpot, aprovechando todas las capacidades incluidas dentro del plan Content Enterprise. El objetivo principal era convertir a la compañía en una organización autosuficiente en el diseño, publicación y mantenimiento de su entorno web corporativo, reduciendo la dependencia de proveedores externos y aprovechando: •

El hosting integrado de HubSpot.

•

Las capacidades de automatización del CRM.

•

Las funcionalidades de HubSpot Content.

•

Las nuevas herramientas de IA integradas en la plataforma.

•

El potencial de personalización y escalabilidad del ecosistema HubSpot.

El proyecto no solo suponía un cambio tecnológico, sino también una oportunidad para rediseñar procesos internos y mejorar la experiencia de trabajo de distintos departamentos. Uno de los principales retos detectados fue la integración del área de selección y gestión de candidaturas del departamento de Recursos Humanos.

2. Situación inicial Antes de la implementación de este sistema, la publicación de ofertas de empleo en isaval.es se realizaba mediante WordPress. El flujo de trabajo era completamente manual: 1. El equipo de RRHH publicaba una nueva entrada en la web. 2. Las ofertas se construían reutilizando contenido anterior. 3. Se utilizaba un formulario de contacto (Contact Form 7) para suscribirse. 4. Las candidaturas llegaban únicamente por correo electrónico a RRHH. 5. No existía un CRM asociado. 6. No había trazabilidad del candidato. 7. No existía pipeline de selección.

8. Toda la gestión dependía de tareas manuales. 9. La gestión de la bolsa de candidatos era manual y compleja. Este sistema presentaba diferentes limitaciones: •

Falta de centralización.

•

Ausencia de automatización.

•

Baja trazabilidad.

•

Pérdida de información contextual.

•

Dificultad para escalar.

•

Alto componente manual.

•

Dependencia técnica externa.

El cambio de plataforma suponía un reto importante para RRHH, ya que el nuevo entorno debía mantener la sencillez operativa y, al mismo tiempo, mejorar radicalmente el sistema. El reto para marketing era producir un espacio que, en lugar de resultar excluyente para sus perfiles, aprovechara la inercia para mejorar sus procesos con visión y escalabilidad.

3. Objetivo del proyecto El objetivo fue diseñar un sistema integrado dentro de HubSpot capaz de: •

Publicar ofertas de empleo dinámicas.

•

Gestionar candidaturas automáticamente.

•

Crear un sistema escalable.

•

Centralizar toda la información en el CRM.

•

Automatizar procesos repetitivos.

•

Mejorar la trazabilidad del candidato.

•

Facilitar el trabajo del departamento de RRHH.

•

Aprovechar el potencial de la IA como apoyo técnico y funcional.

•

Crear una capa de privacidad que proteja su trabajo y datos.

Además, el proyecto debía cumplir otro objetivo clave: Transformar un problema derivado del cambio de plataforma en una oportunidad real de mejora operativa.

5. Desarrollo de la solución 5.1 Construcción del nuevo entorno web El nuevo sitio web corporativo de Isaval Global se ha desarrollado íntegramente dentro de HubSpot. Para ello: •

Se ha utilizado una plantilla base gratuita.

•

Se han desarrollado módulos personalizados para funcionalidades avanzadas.

•

Se han creado componentes dinámicos.

•

Se ha integrado HubDB como base de datos de publicación.

•

Se ha aprovechado el sistema de hosting de HubSpot.

El objetivo era conseguir un entorno: •

Autónomo.

•

Escalable.

•

Gestionable internamente.

•

Flexible.

•

Conectado al CRM.

5.2 Sistema dinámico de ofertas de empleo Uno de los desarrollos principales ha sido el sistema de publicación dinámica de ofertas de empleo. Para ello se desarrollaron módulos específicos: Módulo de listado de ofertas Este módulo permite: •

Mostrar automáticamente las ofertas activas.

•

Filtrar información relevante.

•

Presentar tarjetas dinámicas.

•

Mantener una estructura homogénea.

•

Generar URLs dinámicas.

•

Delegar en el Hub la complejidad de la publicación.

Las ofertas se alimentan mediante HubDB. Módulo de página dinámica de oferta Cada oferta genera automáticamente una página individual dinámica con: •

Información del puesto.

•

Departamento.

•

Modalidad.

•

Ubicación.

•

Jornada.

•

Estado.

•

Requisitos.

•

Responsabilidades.

•

Formulario de inscripción.

Todo ello se construye automáticamente a partir de la información estructurada en HubDB.

6. Sistema de carga y gestión de ofertas Para facilitar el trabajo del departamento de RRHH se diseñó un sistema de publicación extremadamente sencillo. Las ofertas pueden publicarse de dos maneras: Opción 1: Importación mediante Excel Se creó una plantilla tipo con todos los campos necesarios para la oferta: •

Código de oferta.

•

Departamento.

•

Ubicación.

•

Tipo de contrato.

•

Modalidad.

•

Descripción.

•

Requisitos.

•

Responsabilidades.

•

Fechas.

•

El departamento únicamente necesita completar las filas e importar la tabla para añadir ofertas nuevas.

Opción 2: Publicación manual en HubSpot HubSpot permite añadir nuevas líneas de forma sencilla directamente en HubDB. Esto facilita: •

Publicaciones rápidas.

•

Ediciones simples.

•

Actualización inmediata.

•

Gestión autónoma.

7. Automatización de candidaturas La parte más diferencial del proyecto ha sido el diseño del sistema automatizado de gestión de candidaturas. Cuando un candidato accede a una oferta y completa el formulario: El sistema: 1. Lee la información del formulario Captura automáticamente: •

Nombre.

•

Email.

•

Teléfono.

•

CV.

•

Carta de presentación.

•

Información adicional.

2. Lee el contexto de la página Además del formulario, el sistema identifica automáticamente: •

Oferta concreta.

•

URL.

•

Código de oferta.

•

Tipo de puesto.

•

Contexto de navegación.

Esto evita tareas manuales y reduce errores.

3. Crea automáticamente un contacto en el CRM Cada candidatura genera automáticamente un nuevo contacto dentro de HubSpot. De esta forma: •

Toda la información queda centralizada.

•

El candidato pasa a formar parte del ecosistema CRM.

•

Se obtiene trazabilidad completa.

•

Lo asigna a RRHH para aplicar la capa de privacidad.

4. Genera automáticamente un objeto personalizado “CANDIDATURAS” Se diseñó un objeto personalizado específico para este proyecto. Este objeto contiene:

•

Propiedades personalizadas.

•

Estados.

•

Asociación automática con contactos.

•

Pipeline de selección.

•

Automatizaciones.

•

Historial.

El nombre de cada candidatura se genera automáticamente concatenando: “Oferta + Nombre del candidato”. Ejemplo: “Comercial zona norte / Carlos Cuadrado”

5. Inserta automáticamente la candidatura en el pipeline Cada candidatura entra automáticamente en el pipeline de selección en la etapa: “Nueva”. A partir de ahí, RRHH puede mover la candidatura entre distintas etapas: •

Criba.

•

Llamada inicial.

•

Entrevista 1.

•

Entrevista 2.

•

Finalista.

•

Contratada.

•

Descartada.

Esto permite trabajar visualmente y centralizar el proceso completo.
