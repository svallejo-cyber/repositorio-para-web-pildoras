From scattered folders to a shared Rhonatherm project

José Manuel Segundo Santiago · Prescription Team Lead · Rhonatherm / Isaval · April 2026

Core idea: create a web tool that automatically codes Rhonatherm projects and generates their folder structure in OneDrive or SharePoint so documentation stops living across personal machines and inconsistent naming rules.

The point is not only to organize files. It is to build a structured project register that improves collaboration now and enables AI to exploit project data later.

Starting point

Today each prescriber and salesperson may manage projects with their own folder names, local or cloud storage choices and personal criteria. Some documentation lives locally, some in the cloud, and visibility for area managers and leadership is partial or nonexistent.

That creates unstructured information, risk of duplication and excessive dependence on individual memory instead of institutional knowledge.

What the tool proposes

The proposal starts with a unique coding system by branch, province, year and sequence number. On top of that code, the application would register core project fields and generate the project folder structure automatically.

Example code: AOR.MA.2026.001.Reforma vivienda Málaga
Branch · Province · Year · Province sequence · Project name
Single register

Automatic code, client, salesperson, status, amount and notes.

Cloud folders

Automatic creation of the documentation structure in OneDrive or SharePoint.

Filters and consultation

Search by branch, province, status and year.

Export

Excel export for additional analysis.

Immediate collaborative value

If sequence numbers are assigned in real time and folders are created in a shared environment, the business avoids duplicate codes and makes documentation accessible from anywhere with role-based permissions.

The salesperson would keep working in a familiar way, but the information would stop being locked inside one machine.

Relationship with HubSpot

The proposal does not replace the CRM. HubSpot would continue managing contacts, opportunities and commercial follow-up. Rhonatherm Projects would solve a different problem: registering the project, coding it, structuring it and storing its technical documentation.

That complements the current flow without forcing an abrupt behavior change on the commercial team.

Why AI matters here

The document gets the prerequisite right: AI will only create real value once there is a structured historical project base with date, area, salesperson, client, amount and status.

With that base, later phases could enable predictive analysis, internal assistants, intelligent alerts or automated document generation. Without that base, AI would remain generic and much less useful.

Reasonable roadmap

The sequence proposed is coherent:

  1. local pilot of the coding and registration tool,
  2. integration with SharePoint and role-based permissions,
  3. AI layer and advanced analysis,
  4. broader visibility through additional integrations.
Conclusion: before asking AI to think about Rhonatherm projects, this proposal creates the common language and documentation structure that will allow AI to generate real value later.
