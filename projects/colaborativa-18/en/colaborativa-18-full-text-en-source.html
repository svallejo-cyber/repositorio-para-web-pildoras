Structured handling of non-payment case files

Starting need

Non-payment handling starts from a clear need: having an orderly system to manage everything that happens when a customer enters a non-payment situation or exceeds expected payment terms. Follow-up becomes complex because of the variety of possible flows, the documentation involved and the intervention of different people and third parties.

Central element

The proposal places the non-payment case file at the centre of management. The file must gather in an orderly way all information, documentation and actions related to a specific case so that analysis, decisions and actions can be taken with proper judgement.

Initial automation

One of the main triggers for opening a case file should be the receipt of a non-payment communicated by the bank or a payment overrun without apparent justification. At that point, an agent would interpret the bank information, automatically identify the affected customer, the linked remittance and the associated invoices, and leave the file prepared with its initial information.

File content

  • Outstanding invoices and linked delivery notes.
  • Bank return information.
  • Communications with the customer.
  • Internal notes and decisions on how to proceed.
  • Documentation for lawyers, credit insurance or other third parties.

Expected value

The real value is not only in storing documents, but in having full traceability of what was done, who did it, when and what the next step is. This allows any authorised person to understand the case quickly and continue handling it with the full information available.

Conclusion

The project aims to professionalise and organise non-payment handling through a shared methodology, a single control point and a file logic that reduces dispersion, manual work and loss of continuity.
