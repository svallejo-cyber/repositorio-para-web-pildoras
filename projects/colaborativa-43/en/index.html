Collaborative 43 · Transforming fitness-certificate review: from mass checking to automated inconsistency detection

Author: María José Mancebo
María José Mancebo proposes turning fitness-certificate review into a system that validates data, detects mismatches between protocols and job roles, and focuses technical review only on the certificates that really require intervention.

The starting point

Within occupational health and prevention, reviewing fitness certificates is not only about checking whether somebody is fit or unfit. It also requires confirming that identity, role, work center and medical protocols match the preventive reality defined for each position.

The problem is not simply reviewing a high volume. The real issue is reviewing everything in the same way when the value lies in finding inconsistencies fast.

What limits the current model

Volume

Certificates are still reviewed one by one across many people, centers and job profiles.

Judgment

Not every certificate needs the same attention, yet the current model forces a massive, uniform review.

Error

Repetition increases the risk of overlooking data mismatches or incorrectly applied protocols.

Scale

Technical time is absorbed by routine verification instead of the cases that truly require preventive judgment.

The proposal

The opportunity is to turn full manual review into a prior validation and automated inconsistency-detection system. The tool would check core data, role-center alignment and the fit between applied medical protocols and the risks defined for each function.

Instead of reviewing every certificate with the same intensity, the system would flag only incidents, deviations or doubtful cases, leaving technical intervention for where it truly adds value.

How it would work

Base validation

Automatic checking of name, national ID, role, center and minimum certificate consistency.

Detection

Identification of wrong protocols, inconsistent data or deviations from the agreed preventive criteria.

Technical focus

Human review would be reserved for incidents, complex cases or doubts that genuinely require interpretation.

Scalability

The process would absorb more volume with higher reliability and less repetitive effort.

What Isaval would gain

Higher operating efficiency, stronger control consistency, lower human error and better alignment between applied protocols and actual job risks. The preventive function would free time for technical judgment instead of routine mass checking.

Executive conclusion: this pill proposes moving from reviewing every certificate in the same way to automatically detecting where inconsistency really sits, so prevention can work with greater reliability, focus and scale.
