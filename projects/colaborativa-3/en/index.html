From Excel to an OEE dashboard: operational reading of industrial performance

Silvia Soriano · Production Director · Third collaborative contribution to the hub on productivity, visualization and reliable data capture

Context

This case starts from a very concrete objective: turning productivity data extracted from the ERP into a visual tool that makes it easier to understand the real performance of manufacturing plants.

The improvement is not only in showing data, but in turning an operational Excel file into a useful reading of availability, performance and quality.

What has been built

Based on the production and packaging sheets from the ERP, a reading and transformation process was designed to generate an intermediate JavaScript structure that feeds a web dashboard with an MES-like look and feel.

The resulting panel makes it possible to consult a specific plant, change line or machine, and automatically recalculate the content to analyze each section in real time.

What the dashboard adds

Single view

It makes it possible to visualize multiple plants and lines from a single consultation environment.

Key KPIs

It calculates kilos produced, total time, availability, productivity, adjustments and monthly evolution.

Useful analysis

It adds month filters and search by batch or paint, turning visualization into operational analysis.

Scalability

It turns a productivity Excel file into a reusable tool for any section of the plant.

The real critical point

The main bottleneck is not the dashboard. It is the data. To calculate OEE correctly, reliable capture of availability, performance and quality is required, and today that base is still incomplete or irregular in several areas.

  • Availability: planned time, stoppages, breakdowns, waiting periods and unplanned cleaning.
  • Performance: total production, ideal speed, real operating time and weighting by format or family.
  • Quality: good production, waste, rejects, rework and repeated batches.

Main lesson

AI and visualization make it possible to professionalize a task that previously did not scale well, but their impact depends on a prior condition: obtaining the data, structuring it and sustaining its reliability.

Collaborative conclusion: the value is not in building an attractive dashboard, but in using it to raise industrial judgment and make visible where the company needs to improve data capture in order to manage properly.
