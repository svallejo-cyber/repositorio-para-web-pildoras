De Excel a dashboard OEE: lectura operativa del rendimiento industrial

Silvia Soriano · Directora de Producción · Tercera aportación colaborativa del hub sobre productividad, visualización y captura fiable del dato

Contexto

El caso parte de un objetivo muy concreto: transformar los datos de productividad extraídos del ERP en una herramienta visual que permita entender mejor el rendimiento real de las plantas de fabricación.

La mejora no está solo en mostrar datos, sino en convertir un Excel operativo en una lectura útil para analizar disponibilidad, rendimiento y calidad.

Qué se ha construido

A partir de las hojas de producción y envasado del ERP, se diseñó un proceso de lectura y transformación que genera una estructura intermedia en JavaScript para alimentar un dashboard web con estética próxima a un sistema MES.

El panel resultante permite consultar una planta concreta, cambiar de línea o máquina y recalcular automáticamente el contenido para analizar cada sección en tiempo real.

Qué aporta el dashboard

Visión única

Permite visualizar varias plantas y líneas desde un mismo entorno de consulta.

KPIs clave

Calcula kilos fabricados, tiempo total, disponibilidad, productividad, ajustes y evolución mensual.

Análisis útil

Incorpora filtros por mes y búsqueda por lote o pintura para pasar de visualización a análisis operativo.

Escalabilidad

Convierte un Excel de productividad en una herramienta reutilizable para cualquier sección de la planta.

El punto crítico real

El gran cuello de botella no es el dashboard. Es el dato. Para calcular correctamente el OEE hacen falta capturas fiables de disponibilidad, rendimiento y calidad, y hoy esa base todavía es incompleta o irregular en varios puntos.

  • Disponibilidad: tiempos planificados, paradas, averías, esperas y limpiezas no previstas.
  • Rendimiento: producción total, velocidad ideal, tiempo operativo real y ponderación por formato o familia.
  • Calidad: producción buena, mermas, rechazos, retrabajos y lotes rehechos.

Qué aprendizaje deja

La IA y la visualización permiten profesionalizar una tarea que antes no escalaba bien, pero su impacto depende de una condición previa: conseguir el dato, estructurarlo y sostener su fiabilidad.

Conclusión colaborativa: el valor no está en construir un dashboard atractivo, sino en usarlo para elevar el criterio industrial y evidenciar dónde la compañía necesita capturar mejor el dato para gestionar de verdad.
