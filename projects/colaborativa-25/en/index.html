From email threads to a traceable validation flow

María Vallejo · Twenty-fifth collaborative contribution to the hub on packaging validation, approval traceability and AI-built internal tools

The problem

Packaging design validation is still managed by email: heavy PDFs, scattered comments, overlapping versions and an operational flow that depends too much on the person coordinating it.

When production needs to know which design was finally approved, there is no formal traceability showing who approved what, when they did it and on which exact version. The process works, but it works inside Outlook threads and in the manager’s head.

The opportunity was not just to digitize email, but to build a proprietary validation flow with formal traceability, full visibility and orderly access to the approval history.

What has been built

With AI support, María has built a local functional pilot of a web platform for packaging validation. The manager looks up the reference by EAN, uploads the design PDF, assigns reviewers and sets a deadline. Reviewers receive the task in their pending panel, open the design in the browser and leave comments anchored to the exact point on the PDF.

Each reviewer approves or rejects independently. If a rejection happens, a corrected version is uploaded and the tool allows comparison between iterations. When all reviewers approve, the reference automatically moves to Internally validated.

Operational value

Traceability

It records who approved, when they did it and on which exact version of the design.

Review

Comments stay anchored to the PDF itself, without email exchanges describing where the issue is.

Visibility

A traffic-light dashboard shows the real status of every reference and allows filtering by product, status or reviewer.

Scalability

Other teams can access the finally validated design and the full approval history in consultation mode.

What this opens up

  • The pilot already works locally and makes it possible to validate the real flow with the teams involved.
  • The next step is turning the prototype into something deployable in the company environment together with Systems.
  • The biggest value already achieved is not only the software itself, but the precise definition of the process, user profiles and functional needs.
  • The same approach could be replicated in other approval flows currently trapped in email: technical sheets, commercial material or labeling.
Collaborative conclusion: this proposal shows how AI makes it possible to move from an informal email-based process to a tailored internal tool with traceability, visibility and a real basis for scaling complex validation workflows inside Isaval.
