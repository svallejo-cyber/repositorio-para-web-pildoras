5ª píldora · Cómo publico en internet mis píldoras

Proceso real, de extremo a extremo, para convertir ideas en contenido publicado

Punto de partida

El proceso empieza escribiendo yo. El texto bruto es mío al 100%, largo, en lenguaje natural y sin preocuparme inicialmente por la forma.

Primero escribo mucho, luego recorto y vuelvo a escribir. Reviso offline, elimino repeticiones, ajusto el mensaje y solo cuando el texto lo siento mío, paso al siguiente paso.

ChatGPT como primer lector, no como autor

Uso ChatGPT como lector crítico para mejorar claridad, ortografía y estilo. No para sustituir el contenido.

El contenido sigue siendo mío; la herramienta me ayuda a ordenarlo y limpiarlo para que se entienda mejor.

Codex para transformar y publicar

Con el memo final, pido a Codex varias versiones HTML de una sola página, comparo y selecciono la mejor. No hace falta programar: hace falta explicar bien el resultado que se quiere conseguir.

Publicación en modo simple (GitHub + Cloudflare)

1. Iniciar sesión en GitHub y guardar allí los archivos HTML del proyecto.
2. Iniciar sesión en Cloudflare y conectar ese repositorio para publicarlo.
3. Mantener un flujo continuo: cada nueva píldora se añade al repositorio y queda publicada.

Dicho en una línea: en GitHub se guarda el contenido y Cloudflare lo publica.

De páginas sueltas a repositorio vivo

Al principio cada píldora tenía su web independiente. Ahora todo se centraliza en un único hub, con acceso ordenado a todas las piezas y actualización continua.
