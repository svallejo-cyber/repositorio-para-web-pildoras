Pill 5 · How I publish my pills on the web

Real end-to-end process to turn ideas into published content

Starting point

The process starts with me writing. The raw text is 100% mine, long, in natural language, without worrying about form at first.

First I write a lot. Then I cut. Then I rewrite. I review offline, remove repetition, sharpen the message, and only when it feels fully mine do I move forward.

ChatGPT as first reader, not author

I use ChatGPT as a critical first reader to improve clarity, spelling and style. Not to replace content.

The content remains mine; the tool helps me structure and clean it so it reads better.

Codex to transform and publish

With the final memo, I ask Codex for multiple single-page HTML versions, compare them and choose one. You do not need coding skills; you need to explain the expected outcome clearly.

Simple publishing flow (GitHub + Cloudflare)

1. Log in to GitHub and store the HTML files in the repository.
2. Log in to Cloudflare and connect that repository to publish.
3. Keep a continuous flow: each new pill is added and gets published.

In one line: GitHub stores the content and Cloudflare publishes it.

From isolated pages to a living repository

At first each pill had its own page. Now everything is centralized in one hub, with ordered access to every piece and continuous updates.
