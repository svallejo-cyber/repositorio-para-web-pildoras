Pill 4. HOW I PUBLISH MY PILLS ON THE INTERNET

Team,

I want to explain how I am preparing and publishing these pills because I think it also helps to understand how these tools can really be used.

The process starts exactly the same way it always has in my case: writing myself.

The raw text is 100% mine. I write it long, in my natural language, without worrying too much about form. Exactly as I have done many times when I had to write to the company in important moments.

First I write a lot.

Then I cut.

Then I write again.

I review it offline several times. I remove repetition, adjust the message, try to make the idea clear and ensure the text is truly mine. When I feel what I want to say is properly expressed, I move to the next step.

That is where I use ChatGPT.

I have no issue saying it. But I use it as a first critical reader, not as an author. I pass my text and ask whether it is clear, where there is ambiguity, where I repeat myself, or where clarity can improve. It makes small style and spelling corrections and gives me back a cleaner version, but the content remains mine.

With that version, I prepare the internal circular I send to you (as I am doing right now).

From there, something enters the game that I think is already becoming familiar here: Codex.

And here it is worth clarifying something important: Codex is not only an automatic programmer. It can also be used to structure content, transform it into different formats, and leave it ready to be published.

In my case, this is how I use it.

Once I have the final memo, I ask Codex to generate single-page summary HTML versions designed for quick reading. I usually ask for several alternatives, compare which one works best, and keep one.

You do not need to know how to code.

You only need to explain clearly what you want to achieve.

Then I ask Codex to help me with the next step: publishing it.

Here two tools appear that, until literally a week ago, I did not really know either and someone had to explain to me:

GitHub and Cloudflare.

I will explain it very simply.

GitHub is basically a place on the internet where project code is stored. In this case, the code of those HTML pages generated by Codex. It is like an organized archive where the content is saved.

Cloudflare, in turn, is the platform that publishes that content on the web. It connects to the GitHub repository and turns that code into a page accessible from an internet address.

Even more simply:

GitHub stores the content

and Cloudflare publishes it.

And the interesting part is that Codex can help you set up that entire process step by step.

At the beginning each pill had its own individual page. But with Codex’s help I was able to reorganize everything very easily and move from separate pages to a living repository, where all pieces accumulate and can be consulted in an ordered way.

The result is this:

https://repositorio-para-web-pildoras.svallejo-351.workers.dev

That repository is updated every time I add a new pill.

If I had to summarize what I am learning in this process, it would be very simple:

Today one person can think, write, structure content, and publish it on the internet with a speed and autonomy that, not long ago, seemed unthinkable.

And most importantly: without being a programmer.

So... I imagine many of you are already wrestling with Codex these days.

And if some of you are still watching from the sidelines, do not worry too much: the wave is coming anyway.
